# 产品指南

## 品牌与调性
- **专业且亲和：** 既保持作为行业标准规范的严谨性与可靠性，又通过简洁实用的表达贴近普通用户。
- **目标导向：** 所有的沟通与设计都应聚焦于“解决问题”和“提升效率”。

## 文档与写作规范
- **混合型语言风格：** 技能指令（`SKILL.md`）应保持简洁、直接，以任务为中心。在涉及复杂逻辑或架构时，提供深入的技术说明。
- **Markdown 标准化：** 严格遵循 GitHub 风格的 Markdown，保持一致的标题层级，并善用警告（Warning）和提示（Tip）块来突出关键信息。
- **结构化呈现：** 复杂文档必须包含目录索引（TOC），并优先使用 Mermaid 架构图或流程图来直观解释技能工作流。

## 技能开发原则
- **代码质量：** 辅助脚本必须遵循标准化风格指南（如 Python 的 PEP 8），强制要求使用类型提示（Type Hints）以确保代码的可维护性。
- **交互设计：** 遵循“最小干扰原则”。技能应在后台高效执行，仅在需要用户进行关键决策、确认或提供必要输入时才发起交互，从而最大化自动化流程。
- **易用性基准：** 所有的技能必须包含清晰的指令、实际的使用案例和完善的安装/配置指南。
