# Text Style Transfer (文本风格迁移)

此技能用于分析参考文档的写作风格，并根据该风格生成新的内容。它通过分析语气、词汇、句式结构和文档格式，确保生成的内容与参考文档在“神”与“形”上都保持高度一致。

## 使用场景

当用户需要：
1.  **模仿写作风格**：基于一篇现有的文章或文档，创作一篇风格相似的新文章。
2.  **统一文档格式**：确保新生成的项目文档（如 README、技术规范）与旧文档结构一致。
3.  **内容重写**：将一段普通文本重写为特定的风格（如“严谨的学术风格”或“幽默的博客风格”）。

## 工作流程

### 第 1 步：分析风格 (Style Analysis)

首先，你需要分析用户提供的样本文件。

1.  **执行分析脚本**：使用 `scripts/analyze_style.py` 获取文件的结构统计信息。
    ```bash
    python skills/text-style-transfer/scripts/analyze_style.py "path/to/reference_file.md"
    ```
2.  **读取内容样本**：如果文件不是特别巨大，使用 `read_file` 读取文件的全部或部分内容（建议读取前 50-100 行以及有代表性的段落），以直观感受语气和用词。

### 第 2 步：提取特征 (Feature Extraction)

结合脚本输出和直接阅读的内容，总结以下特征：
-   **语气 (Tone)**：例如：正式/非正式、幽默/严肃、简洁/详尽、循循善诱/直截了当。
-   **词汇 (Vocabulary)**：是否有特定领域的术语？使用了什么样的形容词和副词？
-   **句式 (Sentence Structure)**：长短句的分布、被动/主动语态的使用偏好。
-   **结构 (Structure)**：标题层级深度、列表的使用习惯（无序/有序）、代码块的格式、是否有特定的开头/结尾模式。
-   **Markdown 规范**：标题是否加粗？链接的引用方式？图片的展示方式？

### 第 3 步：生成内容 (Content Generation)

根据用户提供的主题、大纲或原始素材，应用提取出的风格特征生成新内容。

-   **应用风格**：在生成过程中，时刻自我检查是否偏离了参考风格。
-   **遵守规范**：严格遵守参考文档的 Markdown 语法规范。

### 第 4 步：一致性检查 (Consistency Check)

生成完成后，自我评估：
-   新文档看起来像是由参考文档的作者写的吗？
-   结构格式是否完全匹配？

## 示例

### 示例 1：基于现有 README 生成新模块文档

**用户输入：**
"请参考 `README.md` 的风格，为新的 'Login' 模块写一个文档，包含功能介绍和使用方法。"

**你的操作：**
1.  运行 `python skills/text-style-transfer/scripts/analyze_style.py README.md`。
2.  读取 `README.md` 内容。
3.  生成 Login 模块文档，模仿 `README.md` 中的标题样式、代码块格式和语言语气。

## 指南

-   **不要** 简单地复制粘贴参考文档的内容。
-   **要** 捕捉“神韵”。如果参考文档喜欢用 emoji，新文档也要用；如果参考文档非常枯燥严谨，新文档就不要讲笑话。
-   如果用户提供的样本文件路径无效，请提示用户检查路径。
