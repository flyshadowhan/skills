# TypeScript 审查检查清单

## 1. 类型安全与推断
- [ ] **避免 `any`：** 是否使用了 `any` 类型？应优先使用更具体的类型、联合类型或 `unknown`（如果类型确实不确定）。
- [ ] **显式返回类型：** 导出函数或公共 API 是否显式定义了返回类型，以提高可读性和契约稳定性？
- [ ] **类型推断：** 是否在简单变量赋值中过度使用了显式类型（冗余类型标注）？
- [ ] **非空断言：** 是否使用了 `!` 断言？应优先使用可选链 (`?.`) 或逻辑判断处理潜在的 `null/undefined`。

## 2. 接口与类型别名
- [ ] **Interface vs Type：** 定义对象结构是否优先使用 `interface`（支持声明合并），定义联合类型、交叉类型或基本类型别名是否使用 `type`？
- [ ] **只读属性：** 对于不应修改的数组或对象属性，是否使用了 `readonly` 关键字 or `ReadonlyArray`？
- [ ] **索引签名：** 是否滥用了索引签名 `[key: string]: any`？应尽量明确属性名称和类型。

## 3. 泛型与工具类型
- [ ] **泛型约束：** 泛型函数或类是否使用了 `extends` 进行必要的类型约束，以增强类型检查？
- [ ] **内置工具类型：** 是否合理使用了 `Partial`, `Pick`, `Omit`, `Record`, `ReturnType` 等工具类型来减少代码重复？

## 4. 枚举与字面量联合类型
- [ ] **字面量联合类型：** 简单的状态或选项是否优先使用字面量联合类型 (`'success' | 'fail'`) 而非枚举（Enum）？
- [ ] **枚举规范：** 如果使用枚举，是否使用了字符串枚举而非默认的数字枚举以增强可读性？

## 5. 类与访问修饰符
- [ ] **私有成员：** 是否正确使用了 `private`（TypeScript 关键字）或 `#`（JavaScript 原生私有字段）？
- [ ] **参数属性：** 是否利用构造函数参数简写属性定义（如 `constructor(private logger: Logger) {}`）？

## 6. 配置与最佳实践
- [ ] **类型转换：** 是否使用了 `as T` 进行不安全的强制转换？应优先使用类型守卫（Type Guards）或 `is` 谓词。
- [ ] **严格模式：** 确保在 `tsconfig.json` 中启用了 `strict: true`，特别是 `strictNullChecks`。
