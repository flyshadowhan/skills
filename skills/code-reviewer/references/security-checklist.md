# 安全审查检查清单

## 1. 输入验证与清理
- [ ] 是否对所有用户提供的输入进行严格的白名单验证？
- [ ] 输入在用于 HTML 前是否清理（XSS 防护）？
- [ ] 输入在用于数据库查询前是否清理（SQL 注入防护）？
- [ ] 输入在用于 OS 命令前是否清理（命令注入防护）？

## 2. 认证与会话管理
- [ ] 密码是否使用强大的现代算法哈希（如 Argon2、bcrypt）？
- [ ] 是否在适当的地方使用多因素认证（MFA）？
- [ ] 会话令牌是否安全处理（HttpOnly、Secure、SameSite cookies）？
- [ ] 是否有安全的登出和会话超时机制？

## 3. 敏感数据处理
- [ ] 代码中是否有硬编码的密钥（API 密钥、密码、令牌）？
- [ ] 敏感数据在静态和传输中是否加密（TLS 1.2+）？
- [ ] 日志是否清除 PII（个人身份信息）和密钥？

## 4. 访问控制（授权）
- [ ] 代码是否遵循最小权限原则？
- [ ] 是否对每个受保护资源在服务器端执行授权检查？
- [ ] 是否有针对不安全直接对象引用（IDOR）的防护？

## 5. 依赖安全
- [ ] 第三方库是否最新？
- [ ] 项目依赖中是否有已知漏洞（CVEs）？

## 6. 错误处理与日志
- [ ] 错误消息是否避免向用户泄露堆栈跟踪或系统信息？
- [ ] 是否记录安全相关信息用于审计（不记录密钥）？

## 7. 常见 Web 漏洞（OWASP Top 10）
- [ ] **注入：** SQL、NoSQL、OS 和 LDAP 注入
- [ ] **认证失效：** 凭据填充、会话劫持
- [ ] **敏感数据泄露：** 缺乏加密、弱密钥
- [ ] **XML 外部实体（XXE）：** 配置不当的 XML 处理器
- [ ] **访问控制失效：** 权限提升
- [ ] **安全配置错误：** 默认账户、冗长错误头
- [ ] **跨站脚本（XSS）：** 反射型、存储型或基于 DOM
- [ ] **不安全反序列化：** 执行来自不可信数据的代码
