# JavaScript 审查检查清单

## 1. 现代语法（ES6+）
- [ ] **箭头函数：** 是否正确使用，特别是关于词法 `this`？
- [ ] **解构：** 是否用于更简洁地访问对象属性和数组元素？
- [ ] **模板字面量：** 是否用于替代字符串拼接？
- [ ] **展开/剩余：** 是否有效用于复制数组/对象或处理可变参数？

## 2. 异步编程
- [ ] **Promise/Async-Await：** 是否处理所有 promise？避免 "Promise 地狱"
- [ ] **错误处理：** `async/await` 是否使用 `try/catch` 块？
- [ ] **并行性：** 多个独立异步任务可并发运行时是否使用 `Promise.all()` 或 `Promise.allSettled()`？

## 3. 作用域和类型
- [ ] **变量声明：** 默认使用 `const`，仅在需要重新赋值时使用 `let`。避免 `var`
- [ ] **相等性：** 使用严格相等（`===`）而不是松散相等（`==`）
- [ ] **类型检查：** 如果不使用 TypeScript，是否对关键输入进行类型检查？

## 4. 代码质量
- [ ] **命名：** 变量和函数使用 camelCase，类使用 PascalCase
- [ ] **模块化：** 函数是否小而专注于单一任务？
- [ ] **全局作用域：** 避免污染全局命名空间
