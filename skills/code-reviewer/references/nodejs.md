# Node.js 审查检查清单

## 1. 运行时与事件循环
- [ ] **阻塞代码：** 避免在 I/O 密集路径中使用同步方法（如 `fs.readFileSync`）
- [ ] **事件循环健康：** 是否有任何长期运行的循环或可能阻塞事件循环的重型计算？

## 2. 资源管理
- [ ] **流：** 处理大型数据集（如文件上传、日志）时是否使用流？
- [ ] **Buffer：** 是否正确处理二进制数据？
- [ ] **内存泄漏：** 检查不断增长的数组、未关闭的事件监听器或持有大型对象的闭包

## 3. 错误处理
- [ ] **全局处理程序：** 是否正确处理 `uncaughtException` 和 `unhandledRejection`？
- [ ] **错误对象：** 始终抛出 `Error` 对象（或子类）以保留堆栈跟踪

## 4. 环境与安全
- [ ] **环境变量：** 配置是否使用环境变量？是否在启动时验证？
- [ ] **包管理：** 检查 `package.json` 是否有不必要或有漏洞的依赖
- [ ] **CORS/头：** Web 服务器是否使用安全头（如 Helmet）？
