# Vue 2 (Options API) 审查检查清单

## 1. Options API 结构与规范
- [ ] **Data 定义：** 组件中的 `data` 是否始终定义为一个返回对象的函数（以防止多个实例共享状态污染）？
- [ ] **Props 验证：** Props 是否显式定义了类型（type）、默认值（default）或验证器（validator）？
- [ ] **计算属性 (Computed)：** 复杂的模板逻辑或需要缓存的计算结果是否使用了 `computed` 而非在模板中直接调用 `methods`？
- [ ] **侦听器 (Watch)：** 是否合理配置了 `immediate: true`（立即执行）或 `deep: true`（深度监听对象）？是否优先考虑使用 `computed` 替代 `watch`？

## 2. 响应式系统限制 (Vue 2 特有)
- [ ] **属性新增：** 动态向对象添加新属性时，是否使用了 `this.$set` 以确保响应式，而非直接赋值？
- [ ] **数组修改：** 修改数组索引或长度时，是否使用了变异方法（如 `push`, `splice`）或 `this.$set`？
- [ ] **初始声明：** 所有需要响应式的属性是否都在 `data` 中预先声明？

## 3. 组件通信与资源管理
- [ ] **单向数据流：** 子组件是否避免了直接修改父组件传入的 Prop？（应通过 `$emit` 触发事件由父组件修改）。
- [ ] **事件总线 (Event Bus)：** 如果使用了全局事件总线，是否在 `beforeDestroy` 钩子中通过 `$off` 解绑了监听器以防止内存泄漏？
- [ ] **资源清理：** 所有的定时器 (`setTimeout`, `setInterval`)、第三方库实例（如图表、地图）或全局 window/dom 监听器是否在 `beforeDestroy` 中正确销毁？

## 4. 生命周期最佳实践
- [ ] **钩子选择：** 初始异步请求是否放在了合适的钩子（通常是 `created` 或 `mounted`）？
- [ ] **DOM 访问：** 是否仅在 `mounted` 及其之后的钩子中访问 DOM？是否优先使用 `ref` 而非 `document.querySelector`？

## 5. 逻辑复用 (Mixins)
- [ ] **命名冲突：** 使用 Mixins 时，是否注意了数据、方法名与组件本体可能发生的命名冲突？
- [ ] **隐式依赖：** Mixins 中是否包含了难以追踪的隐式逻辑？（建议在文档或注释中说明 Mixins 的预期接口）。

## 6. 指令与性能优化
- [ ] **v-for Key：** 循环渲染是否提供了稳定的、唯一的 `key`？是否避免了将数组索引 `index` 作为 `key`？
- [ ] **优先级冲突：** 是否在同一元素上同时使用了 `v-if` 和 `v-for`？（Vue 2 中 `v-for` 优先级更高，建议在父级使用 `v-if` 或使用 `computed` 预筛选数据）。
- [ ] **Scoped CSS：** `<style>` 标签是否带有 `scoped` 属性以防止样式污染？
